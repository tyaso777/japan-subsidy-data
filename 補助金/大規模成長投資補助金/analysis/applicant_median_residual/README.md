# 公開指標では弱く見える採択案件：補正後28件の仮説検証

公開企業PDFから比較できる9指標のうち、同じ申請公募回の申請者中央値を下回る指標が60%以上ある採択・交付企業レコードを、定量・定性・制度の三方向から再検証した分析です。

## 重要なデータ訂正

当初の分析対象は29件でしたが、元データの1・2次 `round` が申請公募回ではなく企業PDFの公開・命名バッチを反映していることが判明しました。公式の1次・2次採択一覧の企業名を照合し、旧社名2件を法人番号で確認したところ、181件中60件で回次が逆でした。

正しい申請公募回の中央値を付け直した結果、最終対象は28件です。

- 除外：アサヒセイレン中部、八立製作所
- 追加：浦島観光ホテル

`application_round_crosswalk_1_2.csv` に181件の固定クロスウォーク、`build_application_round_crosswalk.py` に検証条件、`round_correction_impact.csv` に旧29件から補正後28件への影響を保存しています。元の `cases.csv` は変更していません。

回次監査前に分担作成した定性コーディングと読解メモは、誤って現行正本として利用されないよう `legacy_pre_round_audit/` に隔離しています。これらは履歴・根拠追跡用であり、分析入力時に補正後28件へ絞り込みます。現行の企業別定性正本は `qualitative_coding_28.csv` です。

## 分析上の問い

1. 未満判定の企業内集積は、指標ごとの未満率と欠損だけで説明できるか。
2. 回次、中央値境界、近似指標、期間・主体のずれ、企業規模でどこまで説明できるか。
3. 補助金当たり効果、共同申請、地域、金融機関確認書に別の定量的な優位があるか。
4. 全28件の公開PDFに、需要、能力制約、事業転換、供給網、実行準備等の案件固有事実があるか。
5. それらは別の採択企業より28件で特に強いか。
6. 第6次公募の申請設計へ何を安全に持ち込めるか。

不採択企業の個票・申請書・審査点はありません。したがって、採択確率、予測モデル、因果効果、実際の採択理由は推定していません。「採択残差」は、公開9指標だけでは説明できない部分という記述上の呼称です。

## 補正後28件の定義

対象は1～4次の収録採択・交付企業レコードです。No.1、2、7、8、9、10、11、13、14について、企業値を補正した申請公募回の申請者全体・公式中央値と比較します。

```text
観測指標数 >= 5
かつ
申請者中央値未満の指標数 / 観測指標数 >= 0.60
```

No.8、10、13、14には公開値からの推計・近似が含まれます。No.13は値の望ましい方向が一意ではありません。中央値は公式の足切り線・合格点ではありません。

## 結果の読み方

- 補正後28件は判定可能373件の7.5%です。ただし全28件に少なくとも1つの申請者中央値以上指標があり、「全面的に劣る企業群」ではありません。
- 5%以上の差だけを未満と数えると20件、10%以上では14件です。個社所属の一部は中央値境界に依存します。
- 置換上の期待14.30件に対して28件ですが、置換は現実の指標間相関を壊します。特別な企業群や採択理由を証明する検定ではありません。
- 基準従業員数は小さい傾向がありますが、No.8・10が人数に依存するため、抽出規則自体が小規模企業を選びやすい内生性があります。
- 共同申請等は7/28件対その他10/345件で関連しますが、相乗効果・評価主体のずれ・採択効果のどれが機序かは識別できません。
- 全28件に具体的な定性事実があります。しかし別の採択企業にも広く見られ、28件固有の識別力は確認できません。
- 第6次への助言は「中央値未満でも定性で通せる」ではありません。公式定義で数値を再計算し、需要→能力制約→投資→効果→実行可能性を一次資料でつなぐことです。
