# 分析方法

## 1. 母集団と比較対象

- 企業別データ：`data/processed/cases.csv` に収録された1～4次の公開企業PDF 381企業レコード。
- 公式比較値：`data/reference/official_round_benchmarks.csv` の1～5次申請者・採択者代表値。
- 品質感度：`analysis_exclusion_recommended=false` の202社を「品質除外なし」として併記。
- 分析単位は申請案件でも公式採択案件でもなく、交付決定後に公開されたPDFから作った企業レコードです。共同申請では一つの案件に複数企業が含まれ得るほか、公開時点・収録単位も異なります。このため企業PDF母集団は公式採択者母集団と件数が一致せず、収録企業レコード数が公式採択案件数を上回る回もあります。
- 企業PDF中央値と公式採択者代表値の比較は、独立した2群の比較でも企業別の照合でもなく、集計水準が大きく外れていないかを確認するものです。「申請者全体」には採択者も含まれます。

「品質除外なし」は正確性を認証するラベルではありません。率定義、代表主体、算術対応などについて、既知の重大な注意フラグがないことだけを意味します。品質注意179社には、率定義の曖昧さ135件、代表主体の曖昧さ56件、事業費本文との数値不一致13件、率の算術不一致8件、売上の算術不一致6件が含まれます（重複あり）。

## 2. 現行9指標

ダッシュボードと同じ次の9指標を使います。

1. No.1 全社年平均売上高成長率
2. No.2 全社売上高増加額
3. No.7 補助事業年平均労働生産性の伸び
4. No.8 補助事業付加価値増加額（簡易補正推計）
5. No.9 年平均従業員目標賃上げ率
6. No.10 従業員給与支給総額の増加額（推計）
7. No.11 年平均役員目標賃上げ率
8. No.13 全社売上高に対する投資額割合（近似）
9. No.14 補助金額に対する付加価値増加額割合（推計）

No.8は、公開PDFの労働生産性×補助事業従業員数で求めた付加価値額の目標・基準差を1.1倍した集計水準補正です。1.1は、分析対象と同じ1～4次の企業PDF粗近似中央値と公式採択者代表値の水準差を参考にした**インサンプル較正**であり、独立データによる妥当性検証ではありません。No.14は、1.1倍する前のNo.8粗近似分子÷補助金額×100です。No.8・14とも企業別公式値ではありません。

No.8とNo.14は分母と表示単位が異なりますが、付加価値増加額の粗近似分子を共有します。したがって、9指標を独立した9票として数えたり、No.8とNo.14を独立した二つの証拠として解釈したりしません。除外・領域集約による感度分析を併記します。

No.13は、公募要領の審査項目が「事業者全体の売上高における設備投資額の比率が高い水準であるか」を問うため、**制度上の評価方向は正**です。一方、1～5次の公式代表値では5回中4回で採択者側が申請者側より低く、過去の集団間識別は単調ではありません。この観察から公式方向を否定せず、現行9指標の結果を主表示した上で、No.13除外8指標を感度仕様として示します。

公式公表資料は原則として中央値ですが、一部指標は平均値です。本分析では両者をまとめて「公式代表値」と呼び、企業PDF再構築値との比較時には元資料の統計量種別を維持します。

## 3. 低位企業レコードの定義

主要定義は次のとおりです。

```text
観測指標数 >= 5
かつ
同回の採択者中央値未満の指標数 / 観測指標数 >= 0.60
```

- ダッシュボード互換：現行9指標で判定。5指標以上を観測できる373企業レコード中118（31.6%）。全381レコードに対しては31.0%。品質除外なしでは判定可能199中58（29.1%）。
- No.13除外感度：8指標で判定。5指標以上を観測できる371企業レコード中113（30.5%）。全381レコードに対しては29.7%。品質除外なしでは判定可能198中55（27.8%）。

該当割合は、全収録レコードを分母にする `lagging_pct` と、最低観測数を満たす判定可能レコードだけを分母にする `conditional_lagging_pct` を分けて出力します。本文で低位割合を解釈するときは後者を優先し、分母を明記します。

最低観測数5は、9指標の過半を観測できない企業を主要判定から外すための情報量条件です。60%は「観測指標の明確な過半が採択者中央値未満」という操作的なスクリーニング閾値であり、制度上の基準でも統計理論から一意に導かれた境界でもありません。この任意性を評価するため、次の感度分析を行います。

最低観測数3・5・7、未満割合50%・60%・2/3・75%・80%、全社・品質除外なしの組合せを `lagging_definition_sensitivity.csv` に出力します。

## 4. 中央値以上観測の領域分解

No.13除外感度の8指標を次の3領域に分けます。

- 成長：No.1、No.2
- 生産性・付加価値：No.7、No.8、No.14
- 賃金：No.9、No.10、No.11

領域内で観測できた指標の一つでも採択者代表値以上（`>=`）なら「中央値以上の観測が残る領域」とします。この表現は採択理由や審査上の補償を意味しません。

- 2領域以上に中央値以上あり：42レコード
- 1領域に中央値以上あり：66レコード
- 観測指標に中央値以上なし：5レコード

公式代表値は丸められているため、同値を勝ちに含める `>=` と、同値を除く厳密な `>` の両方を確認します。厳密な `>` では上記の内訳は39、65、9です。また、現行9指標低位118レコードについて、申請者代表値以上の指標が観測指標の半数以上なのは80、厳密な過半数なのは67、かつ各値が申請者代表値を厳密に上回る指標が過半数なのは58です。「半数以上」と「過半数」、`>=` と `>` を入れ替えません。

この分解は採択理由の推定ではなく、公開指標上に中央値以上の観測が残るかの記述です。そもそも「60%以上が未満」という抽出規則は最大40%の非未満指標を許すため、領域勝ちの多さには定義上の成分も含まれます。

### 指標集合のロバストネス

No.8・14の共有分子とNo.13の過去識別不安定性に対応するため、`metric_set_robustness.csv` で次を比較します。

- 現行9指標：118
- No.13除外8指標：113
- No.14除外8指標：118
- No.8除外8指標：101
- No.13・14除外7指標：134
- 成長・生産性／付加価値・賃金の3領域単位：164
- 絶対効果3指標：176
- 率4指標：128

3領域単位では、各領域の企業値／採択者代表値比の中央値を一票とし、No.14を生産性・付加価値領域から外します。最低2領域を観測し、2領域以上が1未満なら低位とします。全観測値が採択者代表値未満となるケースは、No.13除外8指標では5ですが、No.13・14除外の重複調整仕様では7です。したがって「説明不能な5社」を確定的な企業集合として扱いません。

## 5. 結論の証拠区分

本分析では、根拠と推論の混同を避けるため次の5区分を使います。

1. **A 公式要件**：第6次の申請要件、返還・取消条件。
2. **B 公式評価項目**：第6次の審査基準、評価方向、加点項目。
3. **C 公開データ上の観察**：381企業レコードの操作的判定と、1～5次の公式代表値の記述比較。収録範囲内で再計算できますが、採否の因果効果ではありません。
4. **D 採択案件の記載パターン**：目的抽出44社の目視符号化で反復した記載。採否識別力は示しません。
5. **E 分析者の実務提案**：A～Dと申請実務を接続した提案。公式要件ではありません。

採択確率、因果効果、審査配点は、非採択企業個票と審査得点がないため、いずれの区分でも推定しません。

## 6. 指標間依存の置換検定

公募回×指標ごとの採択者中央値未満比率と欠損位置を維持し、企業間で上下判定を1万回並べ替えます。現行9指標低位の実測118社に対し、置換平均は約98.7社、95%範囲は88～109社です。

この上振れは、No.8・14が同じ分子を持つこと、No.2・8・10が企業規模を共有することなど、企業内で低位判定が束になる相関構造と整合します。採択の未観測理由を直接示す検定ではありません。

なお、現在の観測数構成で各指標が独立に50%の確率で中央値未満になる単純モデルでも期待値は114.6レコードです。実測118はこれに近く、置換検定の上振れを「謎の採択」の証拠として解釈しません。

## 7. 目視レビュー

目視対象44社は次の二群です。

1. 既存40ペアの公開定量低位側で、現行9指標低位にも該当する20社。
2. 既存レビュー外の現行9指標低位・品質除外なし35社から、公募回と業種を分散して選んだ24社。

公開採択概要を、次の6要素について0～3点で符号化します。

| 点 | 意味 |
|---:|---|
| 0 | 記載なし、または抽出不能 |
| 1 | 一般的な記述のみ |
| 2 | 具体的な記述・数値・主体がある |
| 3 | 実名顧客、契約・受注、用地・認証、明確な能力上限など強い具体証拠がある |

6要素は、需要根拠、能力制約、事業・工程の構造転換、地域・供給網、実行確度、政策・戦略整合です。

44社は無作為標本ではなく、低位群を説明するために選んだ目的抽出です。全社が採択企業であり、単独かつ非盲検の評価です。要素の出現率を非採択企業との差、因果効果、審査配点、採択確率として解釈してはいけません。

既存40ペアについては、低位側と高位側の6要素得点を対応比較し、符号検定のp値を6要素でBonferroni補正しました。実行確度だけは未補正p=0.035でしたが、補正後p=0.21で、6要素すべてについて低位側固有の差は確認できませんでした。しかもペアの両側とも採択企業であるため、この帰無結果も採否識別力の検定ではありません。44社の高い出現率は「採択概要によく現れる記載パターン」としてのみ使います。

## 8. 第6次数値フレーム

`round6_numeric_framework.csv` は次を区別します。

- 5次申請者・採択者：公式代表値。
- 1～5次傾向参照：5個の採択者公式集計値にTheil–Sen直線を当て、第6次へ1期外挿した探索値。
- 検討用ストレッチ：需要・能力・財務の逆算に使うシナリオ。

Theil–Sen外挿は、観測点が5公募回しかなく、公募ごとの企業構成・様式・制度条件も同一ではありません。予測性能を検証したモデルではないため、本文の根拠ではなく付録の探索シナリオとしてのみ掲載します。傾向参照とストレッチは、公式足切り、合格水準、推奨約束値ではありません。申請値は、顧客別需要、設備能力、歩留まり、稼働率、価格、採用期間、賃上げ原資を積み上げた実行可能なベースケースから決めます。

## 9. No.8補正の公募回留保チェック

固定1.1倍のNo.8はインサンプル較正なので、追加確認として1公募回を留保し、残る3回の「公式採択者代表値÷粗近似中央値」の中央値を留保回へ適用します。4回の留保誤差率は−5.4%～+13.5%、学習係数は1.153～1.207でした。

これは公募回集計中央値4点に対する水準移送の確認にすぎません。企業別公式値との照合、順位精度、個社誤差分布、将来の第6次への外部妥当性は検証していません。結果は `value_added_leave_one_round_out.csv` に出力します。

`analyze_round6_reassessment.py` は乱数シード `20260719` を固定し、主要件数にアサーションを置いています。`build_round6_reassessment_report.py` はCSVと `summary.json` からスタンドアロンHTMLを生成します。

査読対応で追加した主な成果物は次です。
