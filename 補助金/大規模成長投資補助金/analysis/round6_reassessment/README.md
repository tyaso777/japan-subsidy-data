# 第6次申請に向けた「採択者中央値未満案件」の再分析

現行の `public_metrics_dashboard.html` と同じ9指標を起点に、1～4次の公開企業PDFから作成した381の**企業レコード**を再分類し、1～5次の公式代表値、第6次事前公開資料、既存40ペアの目視評価、新規24社の目視評価を統合した分析です。共同申請等を含むため、381は申請案件数や公式採択案件数ではありません。

最初に [round6_adoption_reassessment_report.html](round6_adoption_reassessment_report.html) を開いてください。数値の意味、ケース検索、第6次の申請設計まで一つの資料にまとめています。

## 結論の要点

- 現行9指標のうち5指標以上を観測できる判定可能レコードは373/381です。そのうち、60%以上が同回の採択者代表値未満だったものは118/373（31.6%）です。全381レコードを分母にした31.0%とは区別します。品質除外なしでは58/199（29.1%）です。
- 118レコード中117は少なくとも1指標で申請者代表値以上でした。「観測指標の半数以上が申請者代表値以上」は80ですが、厳密な過半数は67、さらに同値を勝ちに含めず「代表値を厳密に上回る指標が過半数」とすると58です。丸められた公式代表値との同値を強く解釈しません。
- No.13（投資額／全社売上高）は、公募要領上は高い水準が正方向に評価される指標です。一方、過去5回の申請者・採択者代表値では採択者側が4回低く、集団間の識別が単調ではありません。このため現行9指標の結果を残した上で、No.13除外は感度分析として示します。除外時は113/371（30.5%）、品質除外なし55/198（27.8%）です。
- No.8とNo.14は付加価値増加額の粗近似分子を共有するため、独立した2票として読めません。No.14除外118、No.8除外101、No.13・14除外134、3領域単位164など、仕様で件数と企業集合が変わります。全観測値が未満のケースも、8指標仕様の5から重複調整仕様では7へ変わるため、固定的な「例外5社」とは扱いません。
- 現行低位44社の目視では、事業・工程の構造転換44/44、能力制約42/44、実行確度42/44、政策整合42/44、地域・供給網40/44、具体的需要33/44が2点以上でした。ただし採択企業のみを目的抽出した単独非盲検評価であり、因果効果や採否識別力ではありません。既存40ペアでも、6要素の多重比較補正後に低位側固有の差は確認できませんでした。
- 第6次では、中央値を機械的に超えることより、需要→能力制約→投資仕様→数量・売上→付加価値・賃金→地域波及→国費当たり効果を同じ数量単位で接続し、足下の全社賃上げ、資金確度、モニタリング、補助金の必要性まで証拠化することが重要です。

## 再現方法

このフォルダで次を実行します。

```powershell
python analyze_round6_reassessment.py
python build_round6_reassessment_report.py
python validate_round6_reassessment.py
```

分析定義の詳細は [methodology.md](methodology.md) を参照してください。

## 重要な注意

非採択企業の個票と審査項目別得点は公開されていません。この分析から採択確率、因果効果、審査配点は推定できません。No.8の1.1倍は同じ1～4次の集計中央値を参考にしたインサンプル較正で、企業別精度の検証ではありません。公募回留保チェックも4個の公募回中央値だけを使うため、順位精度や誤差分布を保証しません。Theil–Sen外挿も5個の公式集計値による探索的付録であり、第6次の予測値・合格水準ではありません。また、第6次資料は事前公開版に基づくため、正式公募開始時に[公式ダウンロードページ](https://chukentou-seichotoushi-hojo.jp/download/)との差分確認が必要です。
